# 트러블슈팅 기록 (Troubleshooting Log)

AOD 백엔드에서 발생한 문제와 그 진단·수정 과정을 기록한다. 각 문서는 **증상 → 원인 → 이전(Before) → 이후(After) → 왜(Why) → 개선 효과** 순으로 정리한다.

> 목적: 같은 문제 재발 방지 + "왜 이렇게 고쳤는지"를 미래의 나/팀이 이해하도록.

## 공통 교훈

- **증상 ≠ 원인**: "CORS 에러"(02)는 사실 502였고, "장르 안 뜸"(03)은 사실 크롤 스케줄 누락이었다. 브라우저/프론트 메시지를 액면 그대로 믿지 말고 **서버 응답을 직접 찍어** 확인.
- **설정/문서는 코드를 따라가야 한다**: 모니터링(04)이 망가진 이유는 코드가 안 내보내는 유령 메트릭을 설정이 참조해서였다.
- **stale 데이터 의심**: 데이터가 비어 보이면 코드 버그 전에 "그 데이터가 실제로 수집/갱신되고 있나"부터 확인(03).
